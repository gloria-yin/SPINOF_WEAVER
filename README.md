# 🗝️ 功能简介

* 这是**同系列第三作**，也是目前自由度最高的剧情生成工具。

* 和前两作不同，织梦阁不针对特定类型（副本/案件），而是面向**一切可能的番外剧情空间**——平行时空、机制碰撞、穿越快穿、连载宇宙。
* 可以看作是生成一个你和角色的DLC单元剧情线，或者把之前固定好的“小剧场”拓展成可以自由RP的“大剧场”。

* 生成的不是一段死板文字，而是一份完整的**舞台搭建指南**，包含大纲、事件池、多维走向，供你自由 RP 落地。

**用于：**
* 平行时空 RP（ABO / 哨向 / 赛博朋克 / 修仙……任何世界观切换）
* 深度机制碰撞（花吐症 / 时间循环 / 身体互换 / 变小动物……）
* 穿越 / 快穿 / 回溯支线
* 跨番外联动的连载级"番外宇宙"

**生成内容包括但不限于：**
* 情境快照（初始感官冲击 + 角色待机状态）
* 动态事件池（灵感触发 + 走向爆点）
* 多维走向引擎（解谜探索 / 权谋争夺 / 情感联结 / 成长蜕变）
* 可选任务系统（含奖惩机制与系统吐槽）
* 世界书内置状态栏（需配置相关正则）
* 联动继承（从上一番外读取未解伏笔 / 关系变动 / 关键信物）

# 🧠 生成器与提示词解析

---

* 本章节属于进阶教程，适合想要修改提示词或深度定制生成内容的用户。
* 但因为这个工具个人认为条目比较复杂，并且没什么很强的针对性，大概不太需要修改。

* 生成器由五个模块构成，模块间强相关，修改时需联动检查：

```
MODULE 0（参数注入，你填的配置）
MODULE 1（参考框架库，番外世界观素材池）
MODULE 2（参数引导手册，框架构建逻辑）
MODULE 3（主体框架生成指令，看到的番外大纲）
MODULE 4（输出约束，<spinoff> 包裹规则）
```

## Prompt 架构总览

整个生成器的 Prompt 被拆成 5 个模块：

```
MODULE 0：参数注入（你填的全部配置）
MODULE 1：参考框架库（背景库 / 灵感池 / 风格库 / 系统库 / 互动角色库）
MODULE 2：参数引导手册（框架构建逻辑 / 各参数如何影响输出）
MODULE 3：主体框架生成指令（番外大纲模板）
MODULE 4：输出约束（<spinoff> / <spinoff_info> 包裹规则）
```

## MODULE 1 · 参考框架库（可调整）

这一块是**整个生成器的"番外世界观素材库"**，生成器会从这里抽取风格参照、背景设定和机制原型。

包含：
- **参考作品系谱**：快穿流 / 时空穿越 / 单元剧 / 系统文等各流派代表作，用于内化风格，不直接引用
- **背景库**：近40种时代×世界观组合，每类附可剧情化关键词（如"港风黑帮"对应"堂口规矩 / 双花红棍 / 义气扛刀"）
- **灵感池**：按机制类型分组的番外变量库（体型变化 / 身份互换 / 记忆异常 / 感知超能 / 强制绑定 / 时间循环 / 穿越元叙事 / 关系狗血 / 情境处境 / NSFW灵感）
- **风格基调库**：14种风格及其核心情绪、节奏特征与结局倾向
- **尺度层级库**：4级冲突强度的危机类型与情绪波动说明
- **系统风格库**：甜萌 / 毒舌 / 高冷 / 中二 / 迷糊 / 老好人 / 商人 / 隐身，各有交互特征
- **互动角色库**：按权力差 / 对立冲突 / 情感联结 / 契约绑定 / 身份错位 / 家族血缘 / 特殊功能等维度分类的角色原型，每类附核心张力与典型形态
- **开场机制库 / 退出机制库**：{{user}} 进入和离开番外的触发方式

可按照喜好修改：
- 想加新的番外背景 → 改**背景库**，注意附可剧情化关键词
- 想加新的灵感机制 → 改**灵感池**，注意标注核心张力和氛围关键词
- 想加新的系统风格 → 改**系统风格库**，注意说明互动方式和语言风格示例
- 想加新的角色原型 → 改**互动角色库**，注意标注功能定位

**修改建议：素材库是 AI 内化参考用的，扩充比替换更安全。**

## MODULE 2 · 参数引导手册（最核心的部分）

这是生成器真正的"大脑"，定义了所有参数如何被解读为具体的生成指令。

```mermaid
flowchart TD
    A["🦴 骨架层：决定番外是什么形状"]
    B["剧情侧重\n决定章节权重分配\n背景驱动 / 灵感驱动 / 关系驱动 / 均衡"]
    C["剧情走向\n决定事件池核心形态\n解谜 / 权谋 / 情感 / 成长"]
    D["骨架 = 章节权重 + 事件池 + 结局主轴"]
    E["🧂 调味层：决定番外是什么味道"]
    F["时代背景+世界观\n舞台搭在哪"]
    G["灵感\n番外的变量"]
    H["风格\n节奏与情绪密度"]
    I["脑洞程度\n反转层数与设定尺度"]
    J["NSFW\n事件池与结局倾向"]
    K["📦 番外完整框架"]

    A --> B
    A --> C
    B -- "×" --> D
    C -- "×" --> D
    D --> E
    E --> F & G & H & I & J
    F & G & H & I & J --> K
```

上方的速查图展示了最关键的框架逻辑——参数分为两层，各有职责。

**骨架层：决定番外是什么形状**

两个正交维度叠加构成骨架：

- **剧情侧重** 决定哪些章节被加权展开。背景驱动时，世界背景和情境场是核心；灵感驱动时，灵感机制展开曲线是核心；关系驱动时，角色信息和互动模式是核心。同一套灵感，侧重不同，篇幅分配完全不同。
- **剧情走向** 决定动态事件池里核心事件的形态和爆点类型。解谜探索走信息揭示链，权谋争夺走博弈态势变化，情感联结走关系节拍积累，成长蜕变走试炼-挫败-突破曲线。走向同时决定结局分支的达成主轴。

两个维度先各自定位，再叠加出最终骨架。先选侧重（哪些章节重），再选走向（事件怎么流动）。

**调味层：决定番外是什么味道**

- **时代背景 + 世界观组合** 决定"舞台搭在哪"，影响场景标志物、角色身份坐标和规则容器。同一个灵感（比如花吐症）放在现代都市和明清宅门，写出来的细节完全不同。
- **灵感** 决定事件池里是否有机制专属事件，以及机制如何与关系碰撞。没有灵感时世界正常运转；有灵感时，机制本身就是番外的张力来源。
- **风格** 决定节奏快慢、情绪密度和允许的尺度上限。甜宠糖点密集，虐心情绪累积后爆发，沙雕跳脱梗密——同一组事件在不同风格下，写法和节奏完全不同。
- **脑洞程度** 决定反转层数和设定叠加的大胆程度。低脑洞反转合理可推导；中脑洞有戏剧性但不离谱；高脑洞允许打破第四墙、规则颠覆、多灵感叠加、荒诞超现实。
- **NSFW** 开启后，事件池可从 NSFW 灵感库抽取，任务可包含亲密度推进，结局分支中至少出现一个亲密向结局。关闭时，亲密场景以氛围烘托替代直接描写。

此外，MODULE 2 还定义了以下参数的解读规则：

- **世界观 × 时间线组合模式**：当前世界/当前时间 = 日常番外；当前世界/非当前时间 = 穿越番外；非当前世界/参考当前设定 = AU同人；非当前世界/完全新世界 = 快穿番外。每种模式对世界背景章节的展开深度有不同要求。
- **系统 / 任务设计规则**：有系统有任务时需生成完整系统章节和任务清单（含主线/支线/隐藏/难度分级）；无任务时改为隐性推进机制，以推进信号替代任务下发。任务数量随尺度级别变化。
- **联动模式**：开启时强制生成 [联动锚点] 章节，包含前序番外、前序结局采用、衔接方式、继承要素清单（至少3条，含至少1条未解伏笔和至少1条关系升级/降级）。关闭时该章节完全不输出。
- **参数填写模式**：随机（AI 自由抽取）/ 抽取（严格从选项中随机）/ 固定（AI 不增不减不替换）。
- **语言选项**：不加密 = 全中文；加密 = 西班牙语输出，关键人名/地名/系统名保留中文。

### 修改建议

MODULE 2 与 MODULE 1 的素材库和 MODULE 3 的输出模板强绑定，是最牵一发动全身的模块。

- 想调整某种剧情侧重的章节权重分配 → 修改对应侧重的「呈现方式」段落
- 想加新的剧情走向类型 → 同步修改「剧情走向」定义和 MODULE 3 的动态事件池走向核心事件/爆点格式
- 想修改任务数量规则 → 修改「系统 / 任务设计指南」中各尺度的数量说明
- **建议先完整跑几个番外再改，改之前备份原始 Prompt。**

## MODULE 3 · 主体框架生成指令（看到的番外大纲）

番外输出内容包含：

```
[基础信息]         番外名称 / 标签 / 类型 / 时间线 / 灵感 / 风格 / 尺度
[主线介绍]         起点→转折→结局条件，150字左右
[世界背景]         （当前世界观跳过；非当前世界观完整展开）
[情境场设定]       开场快照 / 初始张力 / 张力自驱机制 / 角色待机画面
[动态事件池]       灵感-背景事件 / 走向核心事件 / 反转事件 / 走向核心爆点
[{{user}}身份信息] 番外身份 / 能力局限 / 记忆状态
[互动角色信息]     番外身份 / 走向功能 / 隐藏侧面 / 核心互动模式
[NPC/功能角色]    代称 / 登场时机 / 立场筹码
[关键场景/地点]   场景功能 / 情绪基调 / 剧情锚点
[关键道具/物件]   功能象征 / 走向意义
[系统设定]         （有系统时生成）
[任务清单]         （有任务时生成）
[隐性推进机制]     （无任务时生成，以推进信号替代任务）
[其他特殊设定]    礼仪称谓 / 特殊体系规则 / 角色OOC边界
[结局分支]         HE/BE/NE/TE/开放，含离开方式与余韵留痕
[联动锚点]         （联动模式开启时强制生成）
```

### 关键设计点

- **动态事件池的核心逻辑**：事件是素材库，不规定发生顺序，AI 根据 {{user}} 实际行为选择激活。走向核心事件是柴火（渐进积累），走向核心爆点是火花（瞬间引爆），两者不重合。
- **张力自驱机制**：保证 {{user}} 不主动推进时情境仍然有压力，靠时间倒计时 / NPC 自主推进 / 环境恶化 / 规则收紧等施压，而非靠 {{user}} 的主动行为。
- **结局达成主轴**：不同走向使用不同主轴——解谜走向以真相掌握程度为主轴，权谋走向以筹码积累为主轴，情感走向以亲密度变化为主轴，成长走向以蜕变程度为主轴。情感亲密度只在情感联结走向下作为主轴，其余走向中降为副线。
- **联动锚点的强制性**：联动模式开启时，[联动锚点] 是必须生成的章节，信息不足时 AI 须根据已有信息自行推断，不允许以任何理由跳过。

### 修改建议

- 想在大纲中增加新的分析维度（比如"角色关系图谱"） → 在 MODULE 3 对应位置增加新章节，同时在 MODULE 2 的相关参数说明中注明何时生成
- 想调整某类事件池的数量上限 → 修改动态事件池「写作速查」中的总数控制说明
- 想改结局分支的格式 → 修改 [结局分支] 的表格列和写作速查
- **核心章节建议不删，可以在「写作速查」处添加风格偏好微调。**
- **欢迎分享你做的好模板！**

## MODULE 4 · 输出约束（别乱改！）

这里非常重要：

```
<spinoff> ... </spinoff>
<spinoff_info> ... </spinoff_info>
```

- 生成器会解析 `<spinoff>` 为番外完整框架正文
- `<spinoff_info>` 为简要摘要，固定中文，与加密选项无关
- 简要摘要包含：番外名称 / 时空定位 / 剧情框架 / 灵感 / 风格 / 尺度 / 主导维度说明 / NSFW / 互动角色 / 联动模式 / 开场钩子

除非你想修改「简要信息」的显示内容，否则不要动这里。标签结构一旦改变，脚本解析会出错。

# ❓ 常见问题

### Q1：推荐用什么模型？
* **gemini 2.5 pro**（最推荐，最稳定，联动模式也扛得住）
* gemini 3.1 pro（可以试试，效果接近）
* **不建议使用 flash 系列**：本次提示词结构较案件推演台更复杂，token 量也更大，flash 模型容易出现结构遵循不稳定 / 模块丢失的情况

### Q2：NSFW 开启后内容没变化？
* 请确认破限词栏已填写，且填写的是对应角色卡 / 模型有效的破限词
* 破限词留空时开关不会生效

### Q3：AI 主线识别不准确？
* 主线识别依赖最近聊天记录，内容越丰富识别越准
* 如果主线剧情还很短，建议先不开启该功能，手动填写背景说明即可

### Q4：联动模式需要先有番外世界书吗？
* 是的，联动模式会从世界书读取已有番外大纲，所以**第一个番外请先关闭联动**，正常生成并注入世界书后，后续番外再开启联动

### Q5：选项太多不知道怎么选？
* 无脑点"随机"！每个分类都有随机选项，随机出来的组合往往比精心挑选的更有惊喜感

### Q6：没有按钮 / 按钮打不开？
* 请在社区给我看报错，最好用电脑端，F12 开发者模式给我看 error
* 也可以先问 AI，本人代码能力也是一坨 😅
